Source Files Prerequisites

The DELTA Forth system can handle plain ASCII source files. They can be created under any operating system using appropriate editors. For the MS-DOS platform you can use Edit or the Norton Editor and for the Windows platform, Notepad is a good choice. Usually, a DELTA Forth source file has the .4th extension, but that may be different, according to your needs.

Comments

In DELTA Forth, just like in any other Forth system, there are ways to introduce comments in source files. A multiline comment is inserted between parenthesis. A single line comment is inserted using the backslash.

Example:

( This is a comment

spread across

multiple lines. )

\ This is a one-line comment

Include files

If, for any reason, you need to include the contents of a file into the currently compiling file, use the LOAD directive (outside words only).

Example:

LOAD 2stack.4th \ Include file for double-wide stack operations

Words

A word definition begins with a colon followed by a space and the name of the word, no longer than 31 characters. The word definition must end with a semicolon, before the end of the file.

Example:

: HELLO

." Hello world!" CR

;

The program begins execution with the word MAIN, which must be defined somewhere in the source code. Not having a MAIN word defined will issue a compiling error.
