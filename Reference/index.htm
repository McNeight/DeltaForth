DELTA Forth Basics

DELTA Forth is a new Forth dialect and is intended to bring the Forth programming language to the Java world. As everyone knows, Forth is a special concept in the world of programmers. It has been ported to several configurations, starting with the old CP/M and Z80 processor and now ending with Java. DELTA Forth is designed with portability in mind. You will be capable of running Forth code in any Java environment. Despite its flexibility, Forth did not prove flexible enough to be ported to new systems such as 32-bit configurations. DELTA Forth is a new dialect of Forth that removes some unnecessary concepts and adds the power of portability. This document assumes that you are familiar with the Forth programming language in general. Changes and new specifications are described here.

The most important change is the way DELTA Forth handles memory: Each memory cell is 32-bit wide and the total amount of memory available is 1048576 cells (4Mb in all). A memory cell can store either an integer value or an address. Unlike Forth, DELTA Forth is a compiled language. This means that the source code is turned into a special format that is used to execute the program and many concepts in Forth have been removed due to this major change. Thus, words like COMPILE and BUILDS or DOES have disappeared. Instead, DELTA Forth introduces a C-style convention, namely the execution begins with the MAIN word. If the source file does not contain the word MAIN, an error is reported and the compiling is halted. There are few things you can do outside words: defining variables, constants and including other source files. If the compiler encounters a number, it will not be placed on the stack but stored for later reference.

DELTA Forth vs. Other Forth Systems

Because DELTA Forth has dropped many concepts found in other Forth systems, it might not be fully compatible at the source level. This is mainly because DELTA Forth uses the compiled code approach, whilst other don't. At least for the moment, this dialect is not suitable for large projects as it lacks many features found in modern programming. With the help of users, this can become a reliable tool for writing platform-independent applications.

Known problems

  • The ?TERMINAL word raises a Java exception when used. This will be fixed in future releases of the DELTA Forth system.
  • The word INTERPRET is believed to misbehave in certain situations.
  • The DELTA Forth system has only been tested under the Windows 95/98 environment. Although written entirely in Java, there is a remote possibility that it might not work under other OSes.
  • Not all words found in standard implementations are included. DELTA Forth is still in the Alpha development stage.
  • DELTA Forth currently does not support vocabularies.

Planned enhancements

  • Built-in support for floating point
  • Support for Forth extensions in external libraries written in Java. This will allow exceptional flexibility when dealing with files, streams, etc.
