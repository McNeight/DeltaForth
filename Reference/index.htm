DELTA Forth Basics

DELTA Forth is a new Forth dialect and is intended to bring the Forth programming language to the Java world. As everyone knows, Forth is a special concept in the world of programmers. It has been ported to several platphorms, starting with the old CP/M and Z80 processor and now ending with Java. DELTA Forth is designed with portability in mind. You will be capable of running Forth code in any Java environment. Despite its flexibility, the original Forth did not prove flexible enough to be ported to new systems such as a 32-bit OS. DELTA Forth is a new dialect of Forth that removes some obsolete concepts and adds the power of portability and extensibility. This document assumes that you are familiar with the Forth programming language in general. Changes and new specifications are described here.

The most important change is the way DELTA Forth handles memory: Each memory cell is 32-bit wide and the total amount of memory available is 1048576 cells (4Mb in all). A memory cell can store either an integer value or an address. Unlike Forth, DELTA Forth is a compiled language. This means that the source code is turned into a special format that is used to execute the program and many concepts in Forth have been removed due to this major change. Thus, words like COMPILE and BUILDS or DOES have disappeared. Instead, DELTA Forth introduces a C-style convention, namely the execution begins with the MAIN word. If the source file does not contain the word MAIN, an error is reported and the compiling is halted. There are few things you can do outside words: defining variables, constants and including other source files. If the compiler encounters a number, it will not be placed on the stack but stored for later reference.

DELTA Forth vs. Other Forth Systems

DELTA Forth has dropped many concepts found in other Forth systems, therefore it might not be fully compatible at source level. This is mainly because DELTA Forth uses the compiled code approach, whilst other don't. At least for the moment, this dialect is not suitable for large projects as it lacks features of modern programming. With user's help, this can become a reliable tool for writing platform-independent applications.

    What's New

  • You can easily place texts in memory using the woord DUMP. It requires that the destination address be on the stack.
  • Added the COUNT primitive. This counts the characters up to the first 0.
  • Added the EXIT primitive. This will exit a word immediately. Please note that the stack is not cleaned.
  • Added the <# #> conversion words.
  • Added support for JAVA libraries.
  • Fixed a parser bug that prevented compiling with JDK 1.2
  • Fixed the INTERPRET routine. Now it behaves as expected.
  • Fixed the multi-line comment parser. (I found serious errors)

Known problems

  • The word ?TERMINAL does not always work correctly.
  • The DELTA Forth system has only been tested under the Windows 95/98 environment. Although written entirely in Java, there is a remote possibility that it might not work under other OSes.
  • Not all words found in standard implementations are included. DELTA Forth is still in the Alpha development stage.
  • DELTA Forth currently does not support vocabularies.

 

Planned enhancements

  • Built-in support for floating point
  • User-definable paths for libraries and source code
  • User-definable Forth addressable space
